### 타입이란?   

타입은 대상의 특징을 정한다. 
- 문자열 타입   
- 숫자 타입    
- 불린 타입   
- null 타입   
- undefined 타입    

이 외는 객체 타입   

기본 타입의 인스턴스를 '값'  
객체 타입의 인스턴스를 '객체'  

자바스크립트에서 타입을 가진 대상은 값과 객체. 변수에 타입은 없다. 변수에 임의의 타입의 값을 대입 하거나 임의 타입의 객체를 참조할 수 있다.  
변수에 타입이 없는 언어를 동적 타입 언어라고 한다.  

자바스크립트에는 언어 사양상 타입을 정의하는 구문이 존재하지 않는다. 특별한 구문을 사용하지 않고, 객체에 프로퍼티나 메서드 등으로 특징을 정의한다.    
그 결과 그것이 객체의 타입이 된다. 타입이란 특징의 공통성이다. 각 객체에 공통적인 특징을 갖게 하기 위해 프로토 타입 객체를 사용한다.    
타입이란 특징의 공통성이다. 각 객체에 공통적인 특징을 갖게 하기 위해 프로토 타입 객체를 사용한다. 이런 스타일을 프로토 타입 기반이라고 한다.

### 내장 타입의 개요  

내장 타입을 다섯 개의 기본 타입과 객체 타입으로 분류한다. 기본 타입이라는 용어를 사용하겠다.   

#### 자바스크립트의 기본 타입  

자스의 문자열 타입은 기본 타입이다. 숫자 타입은 한 종류. 내부 표현은 64비트 부동소수점 수 이다.  
숫자 타입이 한 종류인 것은 자스가 실행 효율보다도 프로그래밍 효율을 우선시 하기 때문이다.  

null 타입은 리터럴 값.  
undefined 타입은 미정의를 의미하는 타입.

### 문자열 타입  

#### 문자열 값 리터럴  
문자열 값은 문자열 리터럴로 표기할 수 있다. 문자열 리터럴은 큰 따옴표(")또는 작은 따옴표(')로 감싼다. 
```
var s = "abc";   //변수 s에 문자열 값을 대입
print(s);      // 변수 s의 값을 표시
abc
```

- 특수 문자 쓸 땐 이스케이프 문자열 사용.  
\n : 개행(LF).  
\t : 텝.  
\b : 벨. 
\r : 개행(CR). 
\f : 피드.  
\v : 수직 탭.  
\\ : 백슬래시.  
\' : 작은따옴표. 
\" : 큰따옴표. 
\xXX : Latin-1 코드의 포인트(X는 0부터 9의 숫자값 또는 2부터 f의 알파벳).  
\uXXX : Unicode 코드의 포인트(X는 0부터 9의 숫자값 또는 2부터 f의 알파벳). 


#### 문자열 타입의 연산  

= 연산자의 우변에 문자열 값을 쓰면 좌변에 값을 대입할 수 있다. 또 우변에 문자열 값을 가진 변수를 쓰면 우변의 문자열 값을 좌변의 변수에 대입할 수 있다.   

- 예시
``` 
js> var s = 'abc';    //문자열 값 'abc'를 변수 S에 대입
js› var s2 = s;     //변수 s의 값을 변수 s2에 대입
js> print (s2);       // 변수 s2의 값은 문자열 값 'abc'

abc                   
```

자스의 문자열 타입은 불변 타입이므로 원래 문자열 값을 변경할 수 없다.   

+ 이용하여 문자열 값 연결.  
```
 var s1 = '012';
 var s2 = '345';
 var s3 = s1 + 52; //문자열 값의 연결
 print (s3) ;      //변수 s3의 값은 문자열 값 012345
012345 

```


+= 연산자로 문자열의 연결과 대입을 동시에 함.  
```
js> var s = '012';
js> 5 += '345'; 
js> print (s) ;    11변수 S의 값은 문자열 값 : 012345*
012345
```

typeof 연산자로 값의 타입을 알 수 있다. 문자열 값에 typeof 연산을 하면 "string" 이라는 문자열 값이 반환된다. 
```
js> typeof 'abc';      //문자열 리터럴에 대한 typeof 연산
string
js> var s ='abc';
is> typeof s;     //변수 5의 값에 대한 typeof 연산
string
is> typeof (s) ;  //괄호도 가능
string
js> typeof (typeof (s)) ;   //typeof 연산의 결과는 문자열 값
string
```

#### 문자열 타입의 비교  
두 가지 항등 연산자 === 와 == 가 있다. 각각에 대응하 부정 연산자인 !== 와 != 도 있다.  
=== 와 == 의 차이는 비교할 때 타입 변환을 하느냐이다. ===는 타입 변환을 하지 않고 비교, ===와 == 의 결과에 차이는 없다. 양쪽 모두 문자열의 내용이 일치하는가를 판단한다.   

```
js> var s1 = '012';
js> var s2 = "0';
js> var s3 = s2 + '12';
js> s1 == s3;    //문자열의 내용 일치
true
```

#### 문자열 클래스 (string 클래스)  
자바스크립트의 문자열 타입은 내장 타입 중 하나이다. 내장 타입과는 별도로  자바스크립트에는 문자열 클래스도 존재한다.  
문자열 클래스의 이름은 string 이다. 예를 들어 다음과 같이 문자열 값의 문자 개수를 구할 수 있다.  

```
js> var s = '012';
js> s.length;  //문자열 값의 프로퍼티 접근으로 보인다
3
js> '012' .length; //문자열 리터럴 값의 프로퍼티 접근으로 보인다
3
```

자바스크립트로 '012'.length라고 쓰면 내장 타입인 문자열 값이 암묵적으로 문자열 객체로 형변환되고, 문자열 객체의 length 프로퍼티에 접근  

#### 문자열 객체  
문자열 객체를 명시적으로 생성할 때 new 연산자를 사용한다.  

`var sobj = new string('abc');   //문자열 객체 생성`  

문자열 객체로 암묵적 형변환 할 수 잇기에 new 로 문자열 객체를 사용하는 경우는 없음.  

```
isy var s = sobj + 'def';   //문자열 객체에서 문자열 값으로의 암묵적 형변환
js> print(s);
abcdef
```
암묵적으로 형변환됨. 다만 문자열의 내용이 같아도 참조 객체가 다르면 false.

### 숫자 타입.   

#### 숫자 리터럴   
자스에서 숫자는 대부분 64비트 부동소수점 수이다.   

- 숫자 리터럴 사용 예시  
```
js› var n1 = 1;//변수 n1에 숫자 1을 대입 
js> var n2 = 2;//변수 n2에 숫자 2을 대입
js> n1 + n2;//변수 n1의 값과 변수 n2의 값을 더함

З
```


#### 숫자 클래스 (Number 클래스)  

```
js> (1). toString();  //소수점과 점 연산자를 구별하기 위해 괄호가 필요
1
is> typeof (1). toString(); //숫자에서 문자열로 변환된 것을 확인
string
```

문자열 객체와 마찬가지로 내부적으로는 암묵적으로 숫자 객체가 생성되지만 그 존재를 의식할 필요는 없다.   


#### Number 함수 호출   
Number 함수를 일반 함수처럼 호출하면 숫자를 반환한다. 이를 명시적으로 형변환하는 데 사용할 수 있다. 

```
js> var n1 = Number (1);
js› typeof n1; number    //변수 n1의 값은 숫자
js> n1 == 1;
true

js> n1 === 1;
true
js> var n = Number('1');  //문자열 값에서 숫자로의 명시적인 형변환
js› print (n) ;
1
```

### 불린 타입.    

불린 타입은 논리값 타입이나 진위값 타입 이라고도 한다. 불린 타이에서 얻을 수 있는 값은 true 나 false의 두 값 뿐이다. 그 외의 값은 없음.  
```
js> var flag = true;
js> print(flag) ;
true
js> print (!flag) ;  //true의 부정은 false
false
js› print(! !flag) ;  //부정의 부정은 원래대로 돌아온다
true
```

### null 타입  

본래의 의미는 아무것도 참조하지 않는다는 상태이다. null 타입이 취할 수 있는 값은 null 값 하나 뿐이다. null 값은 리터럴 값이다.  
하지만 null 값의 typeof 연산의 결과는 "object" 이다.  다른 기본 타입의 값은 typeof 연산으로 타입을 판정할 수 있지만 null 타입의 경우는 null 값과 완전 항등 평가(===)할 필요가 있다.   

<hr> 

### undefined 타입  

undefined 타입에서 취할 수 있는 값은 undefined 값 하나이다. undefined 타입의 typeof 연산 결과는 "undefined" 이다.  

undefined 값이 나오는 곳을 정리.    
• 초기화되지 않은 변수의 값    
• 존재하지 않는 프로퍼티의 값   
• 실질 인자를 부여하지 않고 함수를 호출했을 때 함수 안에서 대응하는 인자의 값   
• return 문 자체가 없는, 혹은 return 문에 식이 없는 경우의 함수의 반환값    
• void 연산자의 평가 결과(void 0으로 undefined 값을 얻는 것은 관용구)

### 객체 타입  
기본 타입 이외의 타입은 전부 객체 타입이다. 5가지 기본 타입 외에는 갹체 타입이라고 생각하자.   
```
js> var obj = 0);  //빈 객체를 생성
is> typeof obj;   //obj 변수의 객체(참조)에 대한 typeof 연산
"object"
```

#### 함수 타입  
함수는 객체로 인식하면 된다. 단, 호출할 코드가 있다는 의미에서 함수는 객체 중에서도 특별함.  
자바스크립트의 공식 타입 판정 수단인 typeof 연산의 결과는 함수에 대해 다음과 같이 "function”이라는 문자열이 된다.    

```
js› function f()  {}   //빈 함수
js> typeof f;
"function"
```

### 타입 변환   

자바스크립트에서는 문맥에 따라 자동적으로 형변환이 일어난다. 예를 들어 if 문의 조건식에 넣은 값은 어떤 타입의 값이라도 불린 타입으로 형변환된다.  
식 안에 쓴 값도 연산자에 따라 형변환된다. 예를 들어 어떤 값을 문자열 값과 연결 연산자(+기호)로 연결하려고 하면 어떤 타입의 값이라도 문자열 타입으로 자동으로 형변환된다.   

이 부분에서는 특정 타입에서 다른 타입으로 형변환하는 사례를 보겠다. 각각 명시적인 형변환과 암묵적 형변환 양쪽을 모두 보겠다.  

#### 문자열 값에서 숫자로의 형변환    
문자열 값에서 숫자로 명시적으로 형변환하는 구체적인 예.  

Number 함수, parselnt 함수, parseFloat 함수를 사용하는 것이 정석. 표기가 간단 하다는 측면에서는 Number 함수가 편리하지만   
'100x'처럼 숫자 이외를 포함하는 문자열 값 인 경우, 결과가 NaN이 되므로 주의. parseint 함수와 parseFloat 함수는 숫자 이외의 문자를 무시하므로 '100x를 100으로 변환.  
parseint 함수는 두번째 인자로 변 환 시의 기수(radix)를 지정할 수 있다. 인자를 생략하면 10진수로 가정하고 형변환한다.    

```
js> typeof Number ('100');   //문자열 값 '100' 을 숫자 100으로 전환
number
js> Number ('100x');
NaN
is> parseInt ('100');
100
js> parseInt ('100x'*) ;
100
js> parseInt ('x');
NaN
js> parseInt('ff', 16);
255
js> parseInt('Oxff', 16);
255
js> parseInt ('ff', 10);
NaN
js> parseInt ('0.1');
0
is> parseFloat('0.1');
0.1
```

문자열에서 숫자로의 암묵적 형변환 예시.   
숫자 연산의 피연산자에 문자열 값을 쓰면 암묵적 숫자로 형변환 된다.   

```
js> '100' - 1;  //문자열 값 ' 100' 이 숫자 100으로 변환
99
Js> ’100' - ‘1'; //두 피연산자인 문자열 값이 숫자로 변환
99
js> '100' - ‘’;
//문자열에서 숫자로의 형변환을 목적으로 한 코드. 나중에 나오는 관용구를 참조
100
```


하지만 + 덧셈은 문자열의 연결 연산이 되기 때문에 예상대로 동작하지 않는다.   

#### 불린 타입으로의 형변환  
if 문이나 while 문 등의 조건식에서 암묵적 형변환을 많이 이용한다. 형변환으로 false 이 되는 겂을 열거하면 다음과 같다 다음의 값 이외에는 모두 true.   
- 숫자 0  
- 숫자 NaN. 
- null 값   
- undefined 값  
- 문자열 값 "(빈 문자열 값)  

! 연산은 불린 타입의 피연산자에 대한 논리 부정 연산. 피연산자가 불린 타일이 아니면 불린 타입으로 형변환된다.    
!! 와 같이 두 개를 접치쓰면 부정의 부정이므로 결국 불린 타입으로 형변환하는 기능만 남는다.   

```
js> !!1;
true js> !!'x‘;
trué
js> !!0; 
false
 js>!!";
false
js› !!null;
false
```


불린 타입의 형변환에서 주의해야 할 부분은 객체 타입. 객체 타입을 불린 타입으로 형변환하면 언제나 true가 된다. 그렇기 때문에 다음 코드는 모두 T를 표시.  
예상과 는 다른 방식으로 동작하므로 주의해야 한다.  

```
js> var b = new Boolean (false);
js> if (b) { print('T'); } else { print('F'); }
T

js> var z = new Number (0) ;
js> if (z) { print('T'); } else { print('F'); }
T

js> var s = new String('');
js> if (s) { print('T'); } else { print('F'); }
T
```
